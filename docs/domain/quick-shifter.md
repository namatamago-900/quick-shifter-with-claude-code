# バイクのクイックシフターに関する知識
※一般的なクイックシフターに関する仕組み・知識を記載する。

## 1. クイックシフターの概要
バイクのクイックシフター（Quick Shifter / QS）は、クラッチ操作やアクセルを戻す動作を省略してシフトアップを可能にするシステムである。
通常のシフトアップでは「アクセルを戻す」「クラッチを切る」「ギアを上げる」「クラッチを繋ぐ」「アクセルを開ける」という一連の工程が必要だが、クイックシフターはこれらを電子制御で自動化する。

### 主な役割とメリット
* **タイムロスの削減:** 加速中の駆動力が抜ける時間を最小限（コンマ数秒）に抑え、シームレスな加速を実現する。
* **疲労軽減:** 長距離ツーリングやサーキット走行など、頻繁なシフトチェンジが必要な場面での左手の負担を大幅に減らす。
* **車体の安定:** クラッチ操作に伴う挙動変化（ピッチング）を抑制し、バンク中などの不安定な状況でも安全にシフトチェンジが可能になる。

## 2. センサーについて
クイックシフターを実現するためにはいくつかのセンサーを車体に配置する必要がある。

### センサーの位置
一般的には以下に配置される。([sensors.md](sensors.md))
* **シフトロッド（変速ペダルとエンジンを繋ぐリンク棒）の中間**
  * ライダーがペダルを動かそうとした際の「わずかな動き」や「圧力」を検知するため
* **クラッチレバー付近**
  * ライダーがクラッチレバーを握っているか、握っていないかの状態を検知するため

## 3. 点火カットの仕組み
走行中にギアを切り替えるには、トランスミッションの「ドッグ（噛み合い部分）」にかかっている強い荷重を一瞬抜く必要がある。これを電子制御で行うのが「カット」工程である。

### 制御のプロセス
1.  **入力検知:** 各センサーの状態を検知し、ECU（※本プロジェクトでは無関係）またはArduinoなどのコンピュータへ信号を送る。
2.  **点火停止:** 各センサーの信号をECUやArduinoが受け取って、点火プラグの火花を極めて短時間（30ms〜120ms、デフォルト設定で概ね55ms〜80ms程度）停止させる(点火カット)。
3.  **荷重の消失:** 点火カットされた瞬間に駆動力が一瞬なくなる（エンジンの出力が抜ける）ことで、ギアにかかっていた圧力が消え、ドッグが外れて次のギアへスムーズに移動する。
4.  **復帰:** ギアが切り替わったタイミングで点火を再開し、再び駆動力を発生させる。

## 4. 運用上の注意点とメンテナンス

### 適切な回転域での使用
クイックシフターは「加速状態」や「一定以上の回転数」での使用を前提に設計されている。低回転域やアクセルを閉じている状態で無理に使用すると、ギアボックスに大きなダメージを与える可能性があるためである。
