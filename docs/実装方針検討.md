# 実装方針検討 — Ducati 900SS クイックシフター (Arduino Nano)

## 1. 概要

本ドキュメントは Arduino Nano を使用したクイックシフターの C++ 実装方針をまとめる。

### 基本方針

- `delay()` 関数は**絶対に使用しない**。タイミング制御はすべて `millis()` / `micros()` を用いたステートマシンで実装する
- シフトセンサーのチャタリング対策（Debounce）を厳密に実装し、1回のシフト操作で点火カットが複数回発生しないようにする
- キャブレター車固有のアフターファイアリスクに配慮し、点火カット時間を必要最小限に抑える

---

## 2. ファイル構成

```
src/
├── main.ino                    # メインスケッチ（setup() / loop()）
├── config.h                    # 設定値・定数の定義
├── rpm.h / rpm.cpp             # RPM計測モジュール
├── shift_sensor.h / .cpp       # シフトセンサー（デバウンス処理を含む）
├── clutch_sensor.h / .cpp      # クラッチセンサー
├── ignition_cut.h / .cpp       # 点火カット制御（OUTPUT: D8）
└── quick_shifter.h / .cpp      # クイックシフター本体（ステートマシン）
```

### 各ファイルの責務

#### `config.h`
- ピン定義（D3, D5, D6, D8）
- 各種閾値・タイムアウト定数の一元管理
- キャリブレーション用パラメータ

#### `rpm.h / rpm.cpp`
- D3 の外部割り込み（INT1）で RPM パルスを計測
- パルス間隔から回転数（rpm）を算出
- L型2気筒の不等間隔パルス（クランク1回転=2パルス）に対応
- 公開インターフェース: `uint16_t getRpm()`、`void update()`

#### `shift_sensor.h / .cpp`
- D5 のデジタル入力を読み取り、デバウンス処理を行う
- 公開インターフェース: `bool isShiftRequested()`、`void resetShift()`、`void update()`

#### `clutch_sensor.h / .cpp`
- D6 のデジタル入力を読み取る（ON = クラッチ操作中）
- 公開インターフェース: `bool isClutchEngaged()`、`void update()`

#### `ignition_cut.h / .cpp`
- D8 の HIGH/LOW 制御（HIGH = 点火カット）
- 公開インターフェース: `void startCut(uint32_t durationMs)`、`void update()`、`bool isCutting()`
- カット開始・タイマーセットは `startCut()`、終了判定は `update()` が `millis()` で管理

#### `quick_shifter.h / .cpp`
- システム全体のステートマシンを実装
- 公開インターフェース: `void update()`（`loop()` から毎回呼び出す）

#### `main.ino`
- `setup()`: 割り込み設定、ピンモード初期化
- `loop()`: 各モジュールの `update()` を呼び出すだけ（ブロッキング処理は禁止）

## 3. ステートマシン設計

### 状態一覧

| 状態 | 説明 |
|------|------|
| `STATE_IDLE` | 通常走行中。センサー監視のみ |
| `STATE_SHIFT_PENDING` | シフトセンサーON検知、デバウンス確定待ち |
| `STATE_IGNITION_CUT` | 点火カット実行中 |
| `STATE_COOLDOWN` | カット後のインターバル期間（ダブルカット防止） |

### 状態遷移図

```
                  シフト検知 かつ
                  RPM ≥ 3000 かつ
                  クラッチ未操作
  STATE_IDLE ──────────────────────────→ STATE_SHIFT_PENDING
      ↑                                         │
      │  センサーOFF or RPM < 3000              │ デバウンス確定（5ms以上継続）
      │  (キャンセル)                            ↓
      │                                  STATE_IGNITION_CUT
      │                                         │
      │                                  カット時間経過
      │                                         ↓
      └──────── クールダウン完了 ──── STATE_COOLDOWN
```

### 各状態の動作詳細

#### STATE_IDLE
- シフトセンサー・RPM・クラッチセンサーを監視
- 遷移条件: シフトセンサーON AND RPM ≥ 3,000 rpm AND クラッチ未操作
- クラッチ操作中はクイックシフター不要のため、点火カットを作動させない

#### STATE_SHIFT_PENDING
- シフトセンサーが ON になった時刻を記録
- `DEBOUNCE_MS`（5ms）以上継続して ON の場合のみ有効なシフト操作と判定
- センサーが OFF に戻った場合は `STATE_IDLE` に戻る（チャタリングとみなす）
- デバウンス確定後、`STATE_IGNITION_CUT` へ遷移

#### STATE_IGNITION_CUT
- `ignitionCut.startCut(cutDurationMs)` を呼び出して D8 を HIGH にする
- カット時間は現在の RPM に基づいて決定（§6 参照）
- タイマー計測は `millis()` で行う（`delay()` 禁止）

#### STATE_COOLDOWN
- 点火カット後の最小インターバル（`COOLDOWN_MS` = 500ms）
- 期間中はシフトセンサーの新たな検知を無視する（ダブルカット防止）
- 遷移条件: クールダウン時間経過 → `STATE_IDLE`

## 4. デバウンス設計（チャタリング対策）

### 問題

シフトスイッチはメカニカルスイッチのため、ON/OFF 切り替え時に数ms間の接点バウンスが発生する。これにより1回のシフト操作で複数回の ON 検知が起きると、点火カットが複数回実行されエンジンが失速する。

### 対策方針（2段階）

#### 第1段階: ソフトウェアデバウンス（shift_sensor.cpp）

```cpp
void ShiftSensor::update() {
    bool currentState = (digitalRead(SHIFT_PIN) == LOW); // LOW=ON（GND接続）

    if (currentState && !lastState_) {
        // 立ち上がり検知 → タイムスタンプ記録
        risingEdgeTime_ = millis();
    }

    if (currentState && (millis() - risingEdgeTime_ >= DEBOUNCE_MS)) {
        shiftDetected_ = true; // デバウンス確定
    }

    if (!currentState) {
        shiftDetected_ = false;
    }

    lastState_ = currentState;
}
```

#### 第2段階: クールダウンインターバル

- 1回の点火カット後、`COOLDOWN_MS`（500ms）の間は新たなシフト検知を無効化
- 機械的なバウンスが長引いた場合や、ライダーが誤って2度シフト操作した場合のセーフガード

## 5. RPM 計測設計

### 方式

- D3（INT1）の外部割り込みでピックアップパルスの立ち上がりを検出
- 連続する2パルス間の時間差（`micros()` で計測）から RPM を算出

### L型2気筒の不等間隔パルスへの対応

- 900SS は90°V型2気筒のため、1回転あたり2パルスだが間隔は不等間隔
- 安定した RPM 計算のため、直近2パルス（=クランク約半回転相当）の間隔で計算する簡易方式を採用

```
rpm ≈ 60,000,000 / (pulse_interval_us * 2)
```

> ⚠️ 実測での校正が必要。不等間隔比率は実走行で確認すること（`ignition.md` §2 参照）。

### タイムアウト（エンジン停止・低速検知）

- 最後のパルスから 500ms 経過した場合は RPM = 0 とみなす
- アイドル（1,200 rpm）でのパルス間隔は約 50,000 µs

### RPM スレッショルド

- 3,000 rpm 未満では点火カットを作動させない（`common.md` §4 参照）

## 6. 点火カット時間の設計

### 基本方針

- キャブレター車のため、点火カット中も混合気が供給される → アフターファイアリスク大
- カット時間は必要最小限に（目安: 60〜100ms）
- ギアポジションセンサー非搭載のため、初期実装は RPM ベースの可変カット時間を採用

### RPM ベースの点火カット時間テーブル

| RPM 範囲 | カット時間 | 想定シフト段 |
|----------|-----------|-------------|
| 3,000〜5,000 rpm | 100 ms | 主に1速→2速 |
| 5,000〜7,000 rpm | 80 ms | 主に2速→4速 |
| 7,000〜9,000 rpm | 60 ms | 主に4速→6速 |

> ギア比変化が大きい低速シフト（1→2、変化率71.5%）は荷重抜き時間が長く必要。
> 高速シフト（5→6、変化率89.4%）は短くて足りる。（`gear.md` §2 参照）

### `config.h` での定義例

```cpp
// ピン定義
constexpr uint8_t PIN_RPM_INPUT    = 3;  // D3: RPMパルス入力（INT1）
constexpr uint8_t PIN_SHIFT_SENSOR = 5;  // D5: シフトロッドセンサー
constexpr uint8_t PIN_CLUTCH       = 6;  // D6: クラッチスイッチ
constexpr uint8_t PIN_IGNITION_CUT = 8;  // D8: 点火カット指令

// 点火カット時間（ms）
constexpr uint32_t CUT_DURATION_LOW_RPM  = 100; // 3,000〜5,000 rpm
constexpr uint32_t CUT_DURATION_MID_RPM  = 80;  // 5,000〜7,000 rpm
constexpr uint32_t CUT_DURATION_HIGH_RPM = 60;  // 7,000〜9,000 rpm
constexpr uint32_t MAX_CUT_DURATION_MS   = 200; // フェイルセーフ最大カット時間

// RPM 閾値
constexpr uint16_t RPM_THRESHOLD_MIN  = 3000; // カット作動最低回転数
constexpr uint16_t RPM_THRESHOLD_MID  = 5000;
constexpr uint16_t RPM_THRESHOLD_HIGH = 7000;
constexpr uint32_t RPM_TIMEOUT_US     = 500000; // RPM=0とみなすパルス無信号時間（µs）

// デバウンス・クールダウン
constexpr uint32_t DEBOUNCE_MS  = 5;   // シフトセンサーデバウンス時間
constexpr uint32_t COOLDOWN_MS  = 500; // カット後の再作動禁止時間
```

## 7. `loop()` の構造

`loop()` は非ブロッキングで構成し、各モジュールの `update()` を順次呼び出す。

```cpp
void loop() {
    rpmSensor.update();     // RPM パルスのタイムアウト管理
    shiftSensor.update();   // シフトセンサーのデバウンス処理
    clutchSensor.update();  // クラッチセンサーの読み取り
    ignitionCut.update();   // 点火カットのタイマー管理（カット終了判定）
    quickShifter.update();  // ステートマシンの状態遷移
}
```

- `delay()` の使用は禁止
- すべての時間待機は `millis()` / `micros()` の差分で判定する
- `loop()` は可能な限り高頻度で実行する（目標実行周期: < 1ms）

## 8. 安全設計

### 作動条件（すべての条件を満たした場合のみ点火カット）

1. RPM ≥ 3,000 rpm
2. シフトセンサー ON（デバウンス確定済み）
3. クラッチ未操作（クラッチスイッチ OFF）
4. クールダウン期間外

### フェイルセーフ

- シフトセンサー・クラッチセンサーのピンは `INPUT_PULLUP` で設定する
  - センサーの NO/NC 論理は実車確認後にコードに反映すること
- RPM 信号が途絶えた場合（500ms 以上パルスなし）は RPM = 0 とみなし、点火カット不作動
- `MAX_CUT_DURATION_MS`（200ms）を超えた場合は強制的に点火を復帰させる（ソフトウェアタイムアウト）
