## 2. 指摘事項

### 【指摘1・軽微】`config.h` コメント内のセクション参照が脱落

- **箇所**: `docs/implementation/02-architecture.md` §2 `config.h` の全体像（コードブロック内）
- **旧**:
  ```
  // 出力はアクティブHIGH（§4.2）
  ```
  ```
  //   D5 シフトロッドSW   : 踏み込み = LOW（上方向プッシュ時のみ ON、§5.3 要確認）
  ```
- **新**:
  ```
  // 出力はアクティブHIGH
  ```
  ```
  //   D5 シフトロッドSW   : 踏み込み = LOW（上方向プッシュ時のみ ON、単方向性要確認）
  ```
- **問題**: 旧では「§4.2 → フェイルセーフ方向」「§5.3 → 単方向性確認」へのトレース手段になっていたが、新では参照先が暗黙化されている。
- **推奨対応**: コードコメントは config.h に写経される最終形なので、Markdown リンクではなく次のような **ファイル相対パス参照** に置換するのが望ましい：
  - `// 出力はアクティブHIGH（docs/implementation/04-safety.md §2）`
  - `// 上方向プッシュ時のみ ON（docs/implementation/05-hardware-scope-tuning.md §1.3 要確認）`

### 【指摘2・軽微】チェックリスト表の二重掲載

- **箇所**: 「本実装着手前のドキュメント追対応チェックリスト」（pin_assign.md / sensors.md / ignition.md の3件）が下記2ファイルに **完全に同じ内容で重複** している。

  - `docs/implementation/05-hardware-scope-tuning.md` §1.5

- **補足**: README §5.5 への参照表記が「`05-hardware-scope-tuning.md` §5.5」となっているが、新ファイルでは §1.5 が正しい（新ファイル内ナンバリングは振り直されているため）。**この記述自体が誤リンク**。→ 指摘4 と統合扱い。

- **問題**: obsolute 側には存在しない新規追記。「細分化」の範囲を超えた純増コンテンツ。
